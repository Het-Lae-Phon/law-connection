# Legal Word Tree — PDPA

ต้นไม้คำศัพท์/โครงสร้างกฎหมาย เชื่อมโยงจาก **สิทธิความเป็นส่วนตัวตามรัฐธรรมนูญ**
ลงมาถึง **พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA)** โดยใช้**โครงสร้างของตัวบทกฎหมายเอง**
(หมวด/ส่วน → มาตรา → วรรค → อนุมาตรา) เป็นโครงหลัก แล้วผูกแนวปฏิบัติ
**TDPG4.0 (Thailand Data Protection Guidelines 4.0)** และ**กฎหมายลำดับรองทุกฉบับ**
เข้ากับมาตราที่แต่ละฉบับตีความหรืออาศัยอำนาจ

## โครงสร้างต้นไม้

```
สิทธิในความเป็นส่วนตัว (Right to Privacy)
└── รัฐธรรมนูญแห่งราชอาณาจักรไทย พ.ศ. 2560 มาตรา 32
    └── พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA)
        ├── บททั่วไป (มาตรา 1–7) / หมวด 1–7 / บทเฉพาะกาล — โครงสร้างจริงของกฎหมาย
        │     └── มาตรา X · <ตัวบทย่อ>          ← ทุกมาตราแสดงเนื้อความ
        │           ├── วรรคหนึ่ง · <ตัวบทย่อ>    ← แยกบรรทัดทุกวรรค
        │           │     └── (๑) (๒) … / (ก) (ข) … / นิยาม “…”
        │           ├── TDPG4.0-X <ชื่อบท>       ← แนวปฏิบัติที่ตีความมาตรานี้
        │           │      เช่น ม.37 → TDPG4.0-I การรักษาความมั่นคงปลอดภัย
        │           └── ประกาศฯ / พ.ร.ฎ. ที่ออกตามมาตรานี้
        │                  เช่น ม.4 → พ.ร.ฎ. 2 ฉบับ (กางเป็นรายมาตราได้) · ม.28–29 → ประกาศโอนข้อมูล
        ├── แนวปฏิบัติเฉพาะด้าน ................ TDPG4.0-AI, ID, IT, HR, MS, DA, PO, MD (ตัดขวางหลายมาตรา)
        ├── รายการรอตรวจสอบ (auto-linked, ยังไม่ยืนยัน)
        └── กฎหมายลำดับรองตามประเภท — พระราชกฤษฎีกา (2) / ประกาศ (24) / ระเบียบ (1)
```

การจับคู่ TDPG4.0 ↔ มาตรา และการผูกกฎหมายลำดับรองกับมาตราที่อาศัยอำนาจ
อยู่ใน [`data/law-map.js`](data/law-map.js) (แก้ไขด้วยมือได้ — เป็นการจัดหมวดเชิงบรรณาธิการ
บางรายการผูกกับมาตราที่ใกล้เคียงที่สุดเมื่อฉบับนั้นไม่ระบุมาตราที่อาศัยอำนาจ)

## การใช้งาน

เปิด [`index.html`](index.html) ในเบราว์เซอร์ได้โดยตรง (ไม่ต้องมี build step)
หรือเสิร์ฟแบบ static เช่น `python3 -m http.server` จากโฟลเดอร์นี้

- **คลิกวงกลม** — พับ/กางกิ่ง (วงกลมทึบ = มีกิ่งซ่อนอยู่)
- **คลิกชื่อโหนด** — เปิดแผงรายละเอียด พร้อมลิงก์ไปยังราชกิจจานุเบกษา / PDF ของ สคส. / บท TDPG4.0

ข้อมูลแบ่งเป็นสามไฟล์:

- [`data/word-tree-data.js`](data/word-tree-data.js) — แกนของต้นไม้ (คัดกรองด้วยมือ):
  ราก → รัฐธรรมนูญ ม.32 → พ.ร.บ. + กิ่งแนวปฏิบัติเฉพาะด้านและรายการรอตรวจสอบ
  (โครงสร้างโหนด: `name`, `type`, `detail`, `links[]`, `children[]`)
- [`data/law-map.js`](data/law-map.js) — ชั้นจับคู่ (คัดกรองด้วยมือ): บท TDPG4.0 ↔ มาตรา
  และกฎหมายลำดับรองแต่ละฉบับ ↔ มาตราที่อาศัยอำนาจ
- [`data/act-sections-data.js`](data/act-sections-data.js) — **generated** โครงสร้างตัวบท:
  หมวด/ส่วน → มาตรา (96) → วรรค (203) → อนุมาตรา/นิยาม ของ พ.ร.บ. และรายมาตราของ พ.ร.ฎ.
  ทั้งสองฉบับ + รายการกฎหมายลำดับรองตามประเภท
  สร้างใหม่ได้ด้วย [`scripts/build-sections.py`](scripts/build-sections.py)
  (อ่านจากฐานข้อมูล law-connection; ฉบับ พ.ร.ฎ. 2566 ใช้ OCR จาก PDF ราชกิจจานุเบกษา
  ผ่านเครื่องมือ `scripts/ocr` ของ thai-law-portal) — อย่าแก้ไฟล์นี้ด้วยมือ

เนื้อความรายมาตราและการแบ่งวรรคสกัดอัตโนมัติจากข้อความที่ได้จาก OCR
(นิยามใน ม.6 และ (๑)(๒)/(ก)(ข) แสดงเป็นบรรทัดย่อยของวรรคที่สังกัด ไม่นับเป็นวรรคแยก)
— โปรดตรวจกับต้นฉบับราชกิจจานุเบกษาก่อนอ้างอิงเป็นทางการ

## แหล่งข้อมูล

1. **กฎหมายลำดับรองของ PDPA** — ชุดข้อมูล law-connection, Act id 299
   (`/act/299` ในระบบ) รวม 28 รายการจากราชกิจจานุเบกษา ห้องสมุดกฎหมายกฤษฎีกา และเว็บไซต์ สคส. (PDPC)
   รายการที่เป็นฉบับซ้ำ (ฉบับเว็บ PDPC กับฉบับราชกิจจานุเบกษา, ฉบับแปลอังกฤษ) ถูกรวมเป็นโหนดเดียวโดยเก็บทุกลิงก์ไว้
2. **TDPG4.0 — Thailand Data Protection Guidelines 4.0**, คณะนิติศาสตร์ จุฬาลงกรณ์มหาวิทยาลัย
   <https://www.oa.law.chula.ac.th/tdpg-4-0/> · Open Access, สัญญาอนุญาต CC BY 3.0 TH
   ใช้ผังหัวข้อ (A–S และหมวดเฉพาะด้าน DPO/AI/DA/HR/ID/IT/MD/MS/PO) เป็นโครงในการจัดกิ่ง
   พร้อมลิงก์ตรงไปยังแต่ละบท
3. **รัฐธรรมนูญแห่งราชอาณาจักรไทย พุทธศักราช 2560 มาตรา 32** — ฐานทางรัฐธรรมนูญของสิทธิ
   ในความเป็นส่วนตัวและการคุ้มครองข้อมูลส่วนบุคคล

## หมายเหตุ

- โหนด "รายการรอตรวจสอบ" คือรายการที่ระบบเชื่อมโยงอัตโนมัติแต่ยังไม่ผ่านการยืนยันโดยผู้เชี่ยวชาญ
  (ปัจจุบันมี 1 รายการ: ระเบียบ ก.ล.ต. ซึ่งน่าจะออกตามกฎหมายฉบับอื่น)
- ลิงก์ `http://localhost:3000/...` ชี้ไปยังระบบ law-connection ที่รันในเครื่อง
  หากนำไป deploy ให้แก้เป็นโดเมนจริงของระบบ
